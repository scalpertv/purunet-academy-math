# 초중등 영어 전자북 구조·설계 루틴

작성일: 2026-05-19

## 목적

이 문서는 푸르넷 수학 전자북을 만들면서 정리된 작업 루틴을 바탕으로, 아래 열여섯 가지 영어 전자북을 안정적으로 만들기 위한 구조와 설계 기준을 정리한다.

- 유치원 영어반
- 영어 노래방
- 영어 듣기
- 영어 말하기
- 영어 파닉스 교실
- 영어 읽기
- 초등 영어 교과서
- 초등 교육청 필수 영단어 전자북
- 초등 영문법 전자북
- 영어 쓰기
- 중등 교육청 필수 영단어 전자북
- 중등 영문법 전자북
- 영어 내신 대비반
- 영어 리딩반
- 영어 동화책
- 영어 도서관

목표는 단순 PDF형 전자책이 아니라 학생 학습, 교사 관리, 관리자 승인, 학습 데이터 보존, Cloudflare 서버 DB, 모바일·단일 HTML·Windows 설치 산출물까지 이어지는 운영형 전자북 구조를 만드는 것이다.

## 푸르넷 수학 작업에서 이어받을 핵심 원칙

1. 콘텐츠 DB와 사용자·학습 DB를 분리한다.
2. 소스코드를 업데이트해도 기존 학생, 교사, 학습 기록, 승인 정보는 삭제하지 않는다.
3. DB 변경은 `CREATE TABLE IF NOT EXISTS`, `CREATE INDEX IF NOT EXISTS`, upsert 중심의 additive migration으로만 진행한다.
4. 앱 코드를 수정하면 기본 검증은 `npm run lint`, `npm run verify`, `npm run onefile` 순서로 확인한다.
5. 단일 HTML을 갱신하는 변경이면 단일 HTML, 모바일 `study.html`, 모바일 서비스워커 캐시까지 함께 갱신한다.
6. Windows 설치형 프로그램은 운영 Cloudflare Pages URL을 우선 열고, 실패할 때만 내장 번들로 fallback한다.
7. 작업 전 `checklist.md`와 `context-notes.md`에 계획과 판단 기준을 남기고, 완료 후 검증 결과를 이어서 기록한다.
8. 콘텐츠를 추가할 때는 원본 자료 수, 앱 반영 수, 누락 수, 중복 수를 비교한 검증 보고서를 남긴다.
9. 화면은 실제 학습이 첫 화면에서 바로 가능해야 하며, 마케팅용 랜딩보다 학습·관리 도구 화면을 우선한다.

## 미국·영국 영어교육 전문가 최적화 프레임

이 프로젝트는 한국 초중등 교과 연계형 영어 학습을 유지하면서, 미국식 표준 기반 문해력 교육과 영국식 파닉스·읽기 유창성 교육을 함께 적용하는 구조로 설계한다. 핵심은 영어를 단어장이나 문법 문제집으로만 분리하지 않고, 듣기·말하기·읽기·쓰기·언어지식·문화적 사용 맥락이 서로 순환하도록 만드는 것이다.

| 기준 축 | 참고 기준 | 프로젝트 적용 원칙 |
| --- | --- | --- |
| 미국식 문해력 표준 | [Common Core ELA Standards](https://corestandards.org/english-language-arts-standards/) | 읽기, 쓰기, 말하기·듣기, 언어 영역을 모든 트랙의 공통 성취 축으로 둔다. |
| 미국식 언어 수행 기준 | [ACTFL World-Readiness Standards](https://www.actfl.org/publications/all/world-readiness-standards-learning-languages) | 의사소통, 문화, 연결, 비교, 공동체 관점을 말하기·듣기·읽기 활동에 반영한다. |
| 다국어 학습자 지원 | [WIDA ELD Standards Framework](https://wida.wisc.edu/teach/standards/eld) | 학습자 수준별 언어 기능, 문장틀, 시각 자료, 단계별 산출물을 제공한다. |
| 영국식 영어 교육 기준 | [National Curriculum in England English](https://www.gov.uk/government/publications/national-curriculum-in-england-english-programmes-of-study/national-curriculum-in-england-english-programmes-of-study) | spoken language, word reading, comprehension, spelling, vocabulary, grammar, punctuation을 구조화한다. |
| 영국식 읽기 기초 | [DfE Reading Framework](https://www.gov.uk/government/publications/the-reading-framework-teaching-the-foundations-of-literacy) | 말하기, 이야기, systematic synthetic phonics, 반복 읽기, 읽기 즐거움 문화를 반영한다. |
| 국제 수준 체계 | [CEFR Descriptors](https://www.coe.int/en/web/common-european-framework-reference-languages/cefr-descriptors) | Pre-A1, A1, A2, B1형 `can-do` 문장으로 학습 성취를 표현한다. |

미국 영어교육 전문가 관점에서는 다음 기준을 우선한다.

- 모든 단원은 `목표 성취문`, `수행 과제`, `평가 증거`, `보충 루틴` 순서로 역설계한다.
- 듣기·말하기·읽기·쓰기를 분리해 가르치되, 한 Lesson 안에서는 반드시 두 개 이상 기능을 연결한다.
- 단어는 뜻 암기보다 문장 속 사용, 어휘군, collocation, 문맥 추론, 재사용 빈도를 중심으로 설계한다.
- 문법은 규칙 설명 뒤 문제풀이로 끝내지 않고, 읽기 지문 분석, 말하기 문장틀, 쓰기 첨삭과 연결한다.
- 학생 리포트는 정답률만 보지 않고 `이해`, `정확도`, `유창성`, `표현력`, `자기수정`을 분리해 기록한다.

영국 영어교육 전문가 관점에서는 다음 기준을 우선한다.

- 초기 읽기는 systematic synthetic phonics 순서를 따르며 음소 인식, grapheme-phoneme correspondence, blending, segmenting을 명확히 분리한다.
- 파닉스 단계의 읽기 자료는 배운 음가로 읽을 수 있는 decodable text를 우선한다.
- 읽기 유창성은 정확도, 속도, 억양, 끊어 읽기, 재읽기 기록으로 관리한다.
- spelling은 파닉스, morphology, orthography를 함께 다루며 받아쓰기와 문장 쓰기로 연결한다.
- 도서관과 동화책은 읽기 즐거움, 재독, 낭독, 독후 대화, 독후 쓰기를 만드는 핵심 트랙으로 둔다.

미국식 강점은 표준 기반 성취 관리와 수행 평가이고, 영국식 강점은 초기 문해력과 읽기 습관 형성이다. 이 프로젝트는 두 관점을 경쟁시키지 않고 `기초 음운·파닉스`, `교과 연계`, `실제 의사소통`, `읽기 문화`, `시험 대비`가 한 앱 안에서 연결되게 조직한다.

## TESOL 전문 영어교육 과정 고도화 프레임

TESOL 관점에서 이 프로젝트는 학생용 영어 전자책이 아니라 `학습자 분석 → 언어 분석 → 수업 설계 → 상호작용 → 수행 산출 → 평가·피드백 → 교사 성찰`이 반복되는 영어교육 운영 시스템이어야 한다. 기준은 [TESOL 6 Principles](https://www.the6principles.org/the-principles), [TESOL Standards](https://www.tesol.org/professional-development/publications-and-research/research-and-standards/standards/), [Cambridge CELTA course topics](https://www.cambridgeenglish.org/teaching-english/teaching-qualifications/celta/about-the-celta-course/), [British Council CPD Framework](https://www.teachingenglish.org.uk/professional-development/teacher-educators/educational-context/continuing-professional-development)를 프로젝트용으로 변환해 적용한다.

| TESOL 교육과정 축 | 프로젝트 적용 | 앱·콘텐츠 반영 |
| --- | --- | --- |
| 학습자와 학습 맥락 이해 | 학생의 학년, 영어 노출 경험, 가정 언어 환경, 학교 교과서, 내신 일정, 학습 동기를 기록한다. | 학생 프로필에 `learnerProfile`, `motivation`, `schoolContext`, `homeSupport`, `preferredMode`를 둔다. |
| 언어 분석과 언어 인식 | 단어, 문법, 발음, 담화, 기능 표현을 form, meaning, use, pronunciation으로 분석한다. | 모든 핵심 표현에 `languageAnalysis`, `pronunciationFocus`, `usageContext`, `commonErrors`를 붙인다. |
| 네 기능 수업 설계 | 듣기, 말하기, 읽기, 쓰기를 분리하지 않고 receptive input과 productive output으로 연결한다. | 한 Lesson에 `inputStage`, `guidedPractice`, `communicativeTask`, `outputEvidence`를 둔다. |
| 수업 계획과 자료 설계 | 목표, 단계, 시간, 상호작용, 예상 오류, 보조자료, 대체 활동을 수업안처럼 저장한다. | 콘텐츠 item에 `lessonPlan`, `timing`, `interactionPattern`, `anticipatedProblems`, `solutions`를 둔다. |
| 교수 기술과 전문성 | 교사가 설명자에 머물지 않고 촉진자, 관찰자, 피드백 제공자, 수업 성찰자로 움직인다. | 교사용 화면에 관찰 체크, 즉시 피드백, 수업 후 성찰, 다음 과제 배정 버튼을 둔다. |
| 지속 평가와 적응 수업 | 단원 종료 시험보다 수업 중 evidence를 모아 다음 수업을 바꾼다. | 퀴즈, 녹음, 받아쓰기, 쓰기 초안, 독후활동을 하나의 포트폴리오로 묶는다. |
| 공동체와 전문성 개발 | 교사 혼자 판단하지 않고 교재, 학생 데이터, 학부모 상담, 교사 회의 기록을 연결한다. | 교사 코멘트, 학부모 상담서, 월간 수업 회고, 콘텐츠 개선 메모를 저장한다. |

TESOL 6 Principles는 아래처럼 프로젝트 운영 원칙으로 바꾼다.

| TESOL 6 Principles | 전자북 학습법 변환 | 교사용 확인 질문 |
| --- | --- | --- |
| Know Your Learners | 학생별 수준, 동기, 학교 진도, 파닉스 결손, 학습 습관을 먼저 진단한다. | 이 학생이 왜 이 활동을 해야 하는가. |
| Create Conditions for Language Learning | 실수해도 재시도할 수 있는 녹음, 다시 읽기, 다시 쓰기 구조를 만든다. | 학생이 안전하게 말하고 고쳐볼 장치가 있는가. |
| Design High-Quality Lessons | 모든 활동을 목표, 입력, 연습, 실제 사용, 피드백으로 설계한다. | 오늘 활동이 실제 언어 사용으로 끝나는가. |
| Adapt Lesson Delivery | 정답률, 녹음, 읽기 속도, 쓰기 오류에 따라 난이도와 보조자료를 조정한다. | 지금 이 학생에게 힌트, 모델, 확장 중 무엇이 필요한가. |
| Monitor and Assess Language Development | 결과 점수보다 성장 증거를 누적해 다음 수업으로 연결한다. | 오늘 남은 evidence가 다음 수업 결정을 바꿀 수 있는가. |
| Engage in a Community of Practice | 교사, 학생, 학부모, 콘텐츠 제작자가 같은 리포트를 보며 개선한다. | 교사 판단과 학부모 설명이 같은 데이터로 연결되는가. |

## TESOL 수업 설계 루틴

모든 트랙의 한 Lesson은 TESOL형 `diagnose → prepare → notice → practice → communicate → feedback → reflect` 루틴으로 설계한다.

| 단계 | 학생 화면 | 교사 화면 | 저장 데이터 |
| --- | --- | --- | --- |
| Diagnose | 짧은 진단, 사전 질문, 레벨 체크 | 현재 수준, 결손, 목표 추천 | `diagnosticResult`, `learnerNeed`, `recommendedTrack` |
| Prepare | 그림, 상황, 핵심 어휘, 배경지식 | 선행 지식, 예상 난점 | `schemaSupport`, `preTeachVocabulary`, `anticipatedProblems` |
| Notice | 예문, 음성, 본문, 대화에서 목표 언어를 발견 | form, meaning, use, pronunciation 표시 | `targetLanguage`, `noticingPrompt`, `languageAnalysis` |
| Practice | 카드, 배열, 따라 말하기, 받아쓰기, 통제 연습 | 정확도, 반복 오류, 힌트 사용 | `controlledPracticeScore`, `hintUse`, `errorPattern` |
| Communicate | 역할극, 발표, 요약, 글쓰기, 독후 말하기 | 실제 사용 여부, 상호작용 패턴 | `communicativeTask`, `interactionPattern`, `outputEvidence` |
| Feedback | 즉시 채점, 녹음 비교, 첨삭, 정답 근거 | 오류 교정 방식, 재도전 과제 | `feedbackMode`, `teacherFeedback`, `retryTask` |
| Reflect | 오늘 배운 표현, 자신감 체크, 복습 예약 | 다음 수업 추천, 학부모 메모 | `studentReflection`, `teacherReflection`, `nextAction` |

이 루틴은 트랙별로 다르게 보이지만 내부 구조는 같아야 한다. 예를 들어 파닉스는 `Notice`가 음가 발견이고, 리딩반은 지문 구조 발견이며, 내신 대비반은 본문 속 시험 포인트 발견이다.

## 언어 분석 고도화 기준

TESOL식 언어 분석은 문법명이나 뜻풀이만으로 끝나지 않는다. 모든 핵심 언어 항목은 교사가 수업 전에 설명할 수 있을 정도로 분석되어야 한다.

| 분석 항목 | 설명 | 예시 저장값 |
| --- | --- | --- |
| `form` | 단어 형태, 문장 구조, 어순, 철자, 축약형 | `Can I have ...?`, `subject + verb + object` |
| `meaning` | 한국어 뜻이 아니라 의사소통 기능과 의미 차이 | 요청하기, 허락 구하기, 과거 경험 말하기 |
| `use` | 언제, 누구에게, 어떤 상황에서 쓰는지 | 친구와 대화, 교실 요청, 발표 도입 |
| `pronunciation` | 강세, 억양, 연결, 약화, 음소 | sentence stress, rising intonation, /æ/ |
| `appropriacy` | 격식성, 공손도, 문화적 주의점 | 친구에게 자연스러움, 선생님에게는 더 공손한 표현 필요 |
| `commonErrors` | 한국 학생이 자주 틀리는 오류 | 어순 오류, 관사 누락, /r/과 /l/ 혼동 |
| `ccq` | 개념 확인 질문 | 내가 지금 부탁하고 있나요, 명령하고 있나요. |
| `modelSentence` | 학생이 바로 따라 쓸 수 있는 모델 | Can I have some water, please. |

문법 트랙은 `form → meaning → use → controlled practice → freer production` 순서로 설계한다. 단어 트랙은 `meaning → form → pronunciation → collocation → sentence use → spaced review` 순서로 설계한다. 말하기와 쓰기 트랙은 정확성만 보지 않고 fluency, complexity, interaction, repair 전략을 함께 본다.

## 기능별 TESOL 학습법

| 기능 | 권장 수업 흐름 | 핵심 평가 | 연결 트랙 |
| --- | --- | --- | --- |
| 듣기 | lead-in, 예측, gist listening, detail listening, transcript noticing, shadowing, response task | 핵심 정보 파악, 세부 정보, 받아쓰기, 섀도잉 개선 | 영어 듣기, 초등 교과서, 내신 대비반 |

## 교사용 TESOL 수업안 템플릿

교사용 화면에서는 각 Lesson을 아래 템플릿으로 열람하고 수정할 수 있어야 한다.

| 수업안 항목 | 입력 내용 |
| --- | --- |
| `mainAim` | 오늘 학생이 할 수 있게 될 핵심 언어 행동 |
| `subsidiaryAim` | 보조 목표, 예를 들어 발음, 어휘, 자신감, 협력 |
| `learnerProfile` | 학생 수준, 결손, 동기, 학교 진도, 최근 오답 |
| `assumptions` | 학생이 이미 알고 있을 것으로 보는 내용 |
| `anticipatedProblems` | 예상 오류, 집중 저하, 발음 혼동, 어휘 부족 |
| `solutions` | 그림 힌트, 문장틀, 추가 모델, 난이도 하향, 짝 활동 |
| `materials` | 음원, 그림카드, 지문, 활동지, 녹음 과제, 루브릭 |
| `procedure` | 단계, 시간, 교사 행동, 학생 행동, 상호작용 패턴 |
| `ccqIcq` | 개념 확인 질문과 활동 지시 확인 질문 |
| `assessmentEvidence` | 퀴즈, 녹음, 받아쓰기, 쓰기 초안, 포트폴리오 |
| `feedbackPlan` | 즉시 교정, 지연 교정, 재진술, 자기수정, 교사 첨삭 |
| `reflection` | 수업 후 잘 된 점, 수정할 점, 다음 수업 조치 |

상호작용 패턴은 `T-S`, `S-S`, `Pair`, `Group`, `Individual`, `Teacher conference`로 저장한다. 유치원과 초등 저학년은 `TPR`, `Choral drill`, `Picture talk`을 추가하고, 중등 내신은 `Text analysis`, `Silent work`, `Teacher feedback`을 추가한다.

## 자기주도 학습 TESOL 루틴

학생이 혼자 학습할 때도 TESOL식 수업 흐름이 사라지면 안 된다. 학생 화면은 아래 순서를 기본값으로 둔다.

1. 오늘 목표를 한 문장으로 확인한다.
2. 그림, 상황, 질문으로 배경지식을 연다.
3. 목표 표현이나 지문을 듣고 읽으며 핵심을 발견한다.
4. 짧은 통제 연습으로 정확성을 만든다.
5. 말하기, 쓰기, 요약, 역할극 중 하나로 실제 사용을 한다.
6. 즉시 피드백과 교사 피드백을 분리해 확인한다.
7. 오늘의 `I can ...` 문장을 체크하고 복습 예약을 한다.

이 구조를 적용하면 전자북은 `문제 풀이 → 채점` 도구가 아니라 `언어 입력 → 언어 발견 → 사용 → 피드백 → 성장 기록` 도구가 된다.

## 전체 학습 조직도

열여섯 개 영어 트랙은 아래 네 개 학습 축으로 조직한다. 학생 화면에서는 학습 축을 탭 또는 사이드 메뉴로 보여주고, 교사 화면에서는 축별 진도와 취약도를 한 번에 볼 수 있게 한다.

| 학습 축 | 포함 트랙 | 교육 목적 | 핵심 산출물 |
| --- | --- | --- | --- |
| 입문·흥미 축 | 유치원 영어반, 영어 노래방, 영어 동화책, 영어 도서관 | 영어 노출량, 소리 친숙도, 이야기 흥미, 읽기 습관을 만든다. | 노래 녹음, 그림카드 반응, 동화 역할극, 읽기 기록장 |
| 소리·문자 축 | 영어 파닉스 교실, 영어 듣기, 영어 말하기, 영어 읽기 | 소리와 글자의 대응, 듣고 따라 말하기, 소리내어 읽기 유창성을 만든다. | 파닉스 통과표, 받아쓰기, 말하기 녹음, 읽기 유창성 점수 |
| 교과·내신 축 | 초등 영어 교과서, 초등 필수 영단어, 초등 영문법, 중등 필수 영단어, 중등 영문법, 영어 내신 대비반 | 학교 수업 예습·복습, 필수 어휘·문법, 교과서 본문 분석, 시험 대비를 담당한다. | 단원 완료율, 어휘 누적 테스트, 문법 오답노트, 본문 분석 노트 |
| 문해력·산출 축 | 영어 리딩반, 영어 쓰기, 영어 도서관, 영어 동화책 | 긴 글 이해, 근거 찾기, 요약, 문장·문단 쓰기, 독후 표현을 강화한다. | 독해 근거 표시, 요약문, 첨삭 기록, 독후활동 포트폴리오 |

학생은 `입문·흥미 → 소리·문자 → 교과·내신 → 문해력·산출` 순서로만 이동하지 않는다. 실제 운영에서는 학생의 학년, 학교 진도, 파닉스 결손, 읽기 습관, 내신 일정에 따라 네 축을 동시에 배합한다.

| 영어 도서관 | 다독과 자기주도 읽기 | 레벨 선택, 음성 듣기, 읽기 기록, 퀴즈, 독후활동 | 읽은 권수, 재독 횟수, 독후 기록, 어휘 확장 | 레벨별 서가와 자유 선택을 함께 제공해 reading for pleasure를 만든다. |

## 미국식·영국식 균형 콘텐츠 메타데이터

콘텐츠 DB에는 미국식과 영국식 차이를 숨기지 않고 명시한다. 학생에게는 혼란을 주지 않도록 기본 표기는 하나로 보여주되, 교사용 화면과 고급 보기에서는 변이를 확인할 수 있게 한다.

| 메타데이터 | 값 | 사용 목적 |
| --- | --- | --- |
| `englishVariant` | `us`, `uk`, `both`, `neutral` | 철자, 발음, 표현 차이를 관리한다. |
| `accentAudioKey` | `us-native`, `uk-native`, `teacher`, `ai-guide` | 같은 표현을 미국식·영국식 발음으로 비교할 수 있게 한다. |
| `spellingPair` | `color/colour`, `center/centre` | 중등 이상 어휘와 쓰기에서 표기 차이를 안내한다. |
| `phonicsSet` | `ssp-phase`, `short-vowel`, `long-vowel`, `digraph`, `trigraph`, `r-controlled` | 파닉스 순서와 decodable text 조건을 연결한다. |
| `communicationMode` | `interpretive`, `interpersonal`, `presentational` | 듣기·읽기, 대화형 말하기, 발표형 산출을 구분한다. |
| `cefrCanDo` | `pre-a1`, `a1`, `a2`, `b1` | 학습 목표와 리포트 성취문을 연결한다. |
| `evidenceType` | `quiz`, `recording`, `dictation`, `writing-draft`, `portfolio`, `teacher-rubric` | 점수 외 학습 증거를 저장한다. |

운영 원칙은 다음과 같다.

- 초등 저학년은 하나의 기본 발음 모델을 먼저 제공하고, 익숙해진 뒤 미국식·영국식 차이를 비교한다.
- 파닉스는 영국식 SSP의 체계성을 가져오되, 한국 학생에게 필요한 단모음·장모음·자음군 보충을 별도 루틴으로 둔다.
- 단어와 문장 예문은 미국식과 영국식 모두 자연스러운 표현인지 검토한다.
- 시험 대비 자료는 한국 학교 교과서와 내신 기준을 우선하되, 원어민 표현 차이는 교사용 주석으로 분리한다.
- 학생 리포트에는 `미국식`, `영국식` 우열 표현을 쓰지 않고 `표기 차이`, `발음 차이`, `사용 맥락`으로 설명한다.

## 전문가 검수와 품질 게이트

모든 콘텐츠는 앱 반영 전에 아래 기준을 통과해야 한다.

| 검수 영역 | 통과 기준 | 실패 시 조치 |
| --- | --- | --- |
| 교육 목표 | 단원 목표가 듣기, 말하기, 읽기, 쓰기, 언어지식 중 최소 하나와 연결되어 있다. | 목표 성취문을 다시 작성한다. |
| 수준 적합성 | 학년, CEFR형 수준, 파닉스 단계, 교과 진도가 서로 모순되지 않는다. | `gradeBand`, `cefrCanDo`, `phonicsSet`을 재분류한다. |
| 저작권 | 교과서 본문, 동화 원문, 노래 가사, 음원, 표지는 사용권을 확인했다. | 전문 저장을 중단하고 메타데이터와 자체 제작 활동문으로 대체한다. |
| 미국·영국 변이 | 철자, 발음, 표현 차이가 필요한 항목에 메타데이터가 있다. | `englishVariant`, `accentAudioKey`, `spellingPair`를 보완한다. |
| 평가 증거 | 학생이 무엇을 했는지 남길 수 있는 evidence type이 있다. | 퀴즈, 녹음, 받아쓰기, 쓰기 초안, 루브릭 중 하나를 연결한다. |
| 접근성 | 모바일 화면에서 글자가 겹치지 않고 음성 없이도 핵심 활동을 이해할 수 있다. | 화면 문구, 버튼 크기, 대체 텍스트, 자막을 보완한다. |
| 교사 운영성 | 교사가 다음 수업 조치를 판단할 수 있는 리포트 지표가 있다. | 취약도, 복습 추천, 코칭 메모 항목을 추가한다. |

## 유치원 영어반 데이터 스키마

유치원 영어반은 문자 학습보다 먼저 듣기, 따라 말하기, 그림 인지, 노래와 몸동작, 생활 표현을 중심으로 저장한다.

| 필드 | 설명 |
| --- | --- |
| `id` | 유치원 영어 활동 고유 ID |
| `theme` | 인사, 색깔, 숫자, 가족, 동물, 음식, 교실, 감정 등 |
| `targetExpression` | 오늘의 핵심 생활영어 표현 |
| `coreVocabulary` | 그림카드와 연결할 핵심 단어 배열 |
| `pictureCardKeys` | 그림카드 이미지 키 |
| `audioKey` | 원어민 음성 또는 TTS 키 |
| `chantKey` | 챈트 또는 리듬 활동 키 |

| `gradeBand` | 권장 학년군 |
| `genre` | 정보문, 이야기, 편지, 대화문, 설명문, 주장문 |
| `theme` | 지문 주제 |
| `passageTitle` | 지문 제목 |
| `passageText` | 지문 본문 |
| `paragraphSummaries` | 문단별 핵심 요약 |
| `keyVocabulary` | 핵심 어휘와 문맥상 뜻 |
| `readingSkills` | 중심 내용, 세부 내용, 추론, 순서, 요약, 목적 파악 |
| `questionSet` | 독해 문항 배열 |
| `answerEvidence` | 정답 근거 문장 또는 문단 |
| `extensionWriting` | 후속 쓰기 또는 말하기 과제 |
| `difficulty` | 기초, 표준, 심화 |

리딩반 학습 엔진은 지문 훑어보기, 핵심 어휘 확인, 문단별 중심 내용 찾기, 세부 내용 확인, 추론, 요약, 근거 표시, 확장 쓰기 과제를 기본으로 둔다.

## 영어 동화책 데이터 스키마

영어 동화책은 한 권의 이야기를 장면 단위로 나누고, 듣기, 따라 읽기, 역할극, 이야기 순서, 독후활동으로 연결한다.

| 필드 | 설명 |
| --- | --- |
| `id` | 동화책 콘텐츠 고유 ID |
| `storybookTitle` | 동화책 제목 |
| `storybookLevel` | 유치원, 초급, 초등 저학년, 초등 고학년 |
| `theme` | 우정, 가족, 모험, 동물, 생활습관, 감정 등 |
| `sceneItems` | 장면별 제목, 이미지 키, 요약, 대사 배열 |
| `readAloudAudioKey` | 읽어주기 음성 키 |
| `characterRoles` | 역할극 인물과 대사 |
| `keyVocabulary` | 장면별 핵심 어휘 |
| `sentencePatterns` | 반복 문장 패턴 |
| `comprehensionQuestions` | 장면 이해 질문 |
| `sequenceActivities` | 이야기 순서 맞추기 활동 |
| `retellingPrompt` | 다시 말하기 또는 그림 설명 과제 |

| `sourcePermissionNote` | 원문, 삽화, 오디오 사용 가능 범위 |

영어 동화책은 영어 도서관의 도서 메타데이터보다 더 학습 활동 중심이다. 동화 원문, 삽화, 오디오는 저작권 자료이므로 정식 라이선스가 있는 자료만 저장하고, 기본은 자체 제작 동화, 장면 요약, 역할극 대사, 독후활동 템플릿으로 구성한다.

## 영어 도서관 데이터 스키마

영어 도서관은 원서, 리더스, 그림책, 단계별 다독 자료를 학습 기록과 연결하는 구조로 관리한다.

| 필드 | 설명 |
| --- | --- |
| `id` | 도서 고유 ID |
| `libraryLevel` | 유치원 그림책, 초급 리더스, 중급 리더스, 중등 리딩 |
| `title` | 도서 제목 |
| `author` | 저자 |

## 화면 구조

첫 화면은 학습자가 바로 공부를 시작할 수 있는 운영형 학습 화면으로 구성한다.

1. 상단 메뉴바.
   관리자 로그인, 교사 로그인, 학생 로그인, 회원가입, 교사 승인, 학생 승인, 학생 등록, 학생 관리, 클래스 관리, 보고서 관리를 메뉴바로 숨긴다.

2. 왼쪽 학습 제어 패널.
   유치원 영어반, 영어 노래방, 영어 듣기, 영어 말하기, 파닉스 교실, 영어 읽기, 초등 영어 교과서, 초등 영단어, 초등 영문법, 영어 쓰기, 중등 영단어, 중등 영문법, 내신 대비반, 리딩반, 영어 동화책, 영어 도서관을 탭이나 세그먼트 컨트롤로 선택한다. 학년군, 난이도, 단원, 복습 모드, 오답 모드도 이 영역에서 선택한다.

3. 중앙 학습 화면.
   유치원 그림카드, 영어 노래방 가사 화면, 듣기 대화, 말하기 녹음 과제, 파닉스 음가 카드, 읽기 유창성 문장, 초등 교과서 단원 활동, 단어 카드, 문법 개념, 쓰기 과제, 교과서 본문 분석, 문장 구조 분석, 리딩 지문, 동화책 장면, 도서 카드, 독서 기록, 문제 풀이, 예문 작성, 오답 풀이가 들어가는 핵심 영역이다. 카드 안에 또 카드를 넣지 않고, 반복 항목이나 문제 단위에만 카드를 쓴다.

4. 오른쪽 또는 하단 결과 패널.
   오늘 학습량, 정답률, 유치원 참여 활동, 노래 따라 부르기 기록, 듣기 이해도, 말하기 녹음 상태, 약한 음가, 읽기 유창성, 블렌딩 취약 패턴, 오답 단어, 쓰기 첨삭 상태, 약한 문법, 내신 취약 본문, 리딩 취약 스킬, 동화책 이해도, 도서 읽기 기록, 추천 복습, 교사용 코칭 메모를 표시한다. 화면 폭이 좁으면 하단 탭으로 내려간다.

5. 교사용 모니터링.
   교사 또는 관리자 로그인에서만 학생별 진도, 오답, 유치원 활동 참여도, 노래 참여도, 듣기 성취도, 말하기 녹음·발표 상태, 음가 인식률, 읽기 유창성, 블렌딩 정확도, 초등 교과서 단원 성취도, 단어 암기율, 쓰기 첨삭 상태, 문법 취약 단원, 교과서별 내신 대비 현황, 리딩 스킬별 성취도, 동화책 독후활동, 도서관 독서량, 일간·주간·월간 리포트를 표시한다.

## 학습 흐름 설계

### 유치원 영어반 기본 흐름

1. 오늘의 주제와 그림카드를 연다.
2. 교사용 진행 멘트와 원어민 음성을 함께 재생한다.
3. 학생은 그림을 보고 단어를 듣고 따라 말한다.
4. 챈트와 몸동작 반응 활동을 진행한다.
5. 색깔, 숫자, 동물, 감정 같은 생활 주제 표현을 짧게 반복한다.
6. 알파벳 놀이 또는 파닉스 준비 활동으로 이어간다.
7. 스티커 보상, 가정 미션, 교사 관찰 메모를 기록한다.
8. 교사용 화면에는 참여도, 따라 말하기 반응, 익숙한 표현, 가정 복습 필요 표현이 표시된다.

### 영어 노래방 기본 흐름

1. 학년군, 주제, 노래 레벨을 선택한다.
2. 노래 제목, 핵심 표현, 핵심 단어를 먼저 확인한다.
3. 가사 하이라이트에 맞춰 듣고 따라 부른다.
4. 느린 속도, 원속도, 반주 모드로 반복한다.
5. 발음, 강세, 리듬 포인트를 짧게 연습한다.
6. 학생 녹음 또는 교사 확인 체크를 남긴다.
7. 빈칸 가사, 표현 찾기, 율동 미션을 진행한다.
8. 교사용 화면에는 참여도, 반복한 노래, 약한 발음 포인트, 추천 복습곡이 표시된다.

### 영어 듣기 기본 흐름

1. 수준, 주제, 음원 유형을 선택한다.
2. 핵심 표현과 듣기 목표를 먼저 확인한다.
3. 전체 음원을 한 번 듣고 큰 흐름을 파악한다.
4. 다시 듣기에서 핵심 정보와 세부 정보를 찾는다.
5. 받아쓰기 또는 빈칸 듣기를 진행한다.
6. 섀도잉으로 짧은 구간을 따라 말한다.
7. 정답 근거가 되는 대본 위치를 확인한다.
8. 교사용 화면에는 듣기 정답률, 받아쓰기 정확도, 놓친 표현, 추천 복습 음원이 표시된다.

### 영어 말하기 기본 흐름

1. 말하기 상황과 역할을 선택한다.
2. 예시 답변과 문장 틀을 확인한다.
3. 따라 말하기로 발음과 억양을 연습한다.
4. 문장 틀을 활용해 자기 답변을 만든다.
5. 역할극 또는 질문·응답을 진행한다.
6. 짧은 발표 또는 의견 말하기를 녹음한다.
7. 교사 피드백과 다시 말하기 과제를 저장한다.
8. 교사용 화면에는 유창성, 발음, 문장 정확성, 자신감 지표가 표시된다.

### 파닉스 기본 흐름

1. 오늘의 글자 또는 철자 패턴을 연다.
2. 알파벳 모양, 대표 음가, 입 모양 가이드를 확인한다.
3. 소리를 듣고 같은 음가를 고른다.
4. 글자와 소리를 연결한 뒤 짧은 음절을 블렌딩한다.
5. 파닉스 단어를 디코딩하고 그림 또는 뜻과 연결한다.
6. 짧은 문장을 읽고 녹음 또는 따라 말하기 상태를 기록한다.
7. 교사용 화면에는 음가 인식률, 블렌딩 정확도, 반복 취약 패턴, 추천 복습 묶음이 표시된다.

### 초등 영어 교과서 기본 흐름

1. 학년, 출판사, 학기, Lesson을 선택한다.
2. 단원 목표와 핵심 의사소통 표현을 확인한다.
3. 듣기 대화와 따라 말하기 활동을 진행한다.
4. 핵심 단어와 표현을 그림카드, 문장 패턴, 짝 활동으로 복습한다.
5. 읽기 지문 또는 단원 글의 핵심 내용을 확인한다.
6. 문장 완성, 짧은 쓰기, 역할극, 교실 게임 활동을 진행한다.
7. 단원 복습 문제와 자기 점검을 완료한다.
8. 교사용 화면에는 단원별 듣기, 말하기, 읽기, 쓰기 성취도와 복습 필요 표현이 표시된다.

### 영어 읽기 기본 흐름

1. 수준과 읽기 텍스트 유형을 선택한다.
2. 모범 읽기 음성을 듣고 끊어 읽기 단위를 확인한다.
3. 학생이 소리내어 읽고 녹음한다.
4. 시간 읽기 목표에 맞춰 다시 읽는다.
5. 발음 집중 단어와 억양 포인트를 확인한다.
6. 짧은 이해 확인 질문을 푼다.
7. 교사 피드백과 다시 읽기 과제를 저장한다.
8. 교사용 화면에는 정확도, 속도, 끊어 읽기, 표현력 지표가 표시된다.

### 영단어 기본 흐름

1. 오늘의 단어 묶음을 연다.
2. 단어 카드로 뜻과 예문을 확인한다.
3. 뜻 맞히기와 철자 입력을 진행한다.
4. 틀린 단어는 즉시 오답 노트에 들어간다.
5. 학습 종료 시 누적 복습 큐가 갱신된다.
6. 교사용 화면에는 암기 완료, 미완료, 반복 오답, 발음·철자 취약 항목이 표시된다.

### 영문법 기본 흐름

1. 개념 카드를 먼저 보여준다.
2. 핵심 패턴을 예문으로 확인한다.
3. 문장 배열 또는 빈칸 문제로 규칙을 적용한다.
4. 오류 수정 문제로 흔한 실수를 잡는다.
5. 단원 확인 퀴즈를 풀고 해설을 본다.
6. 교사용 화면에는 개념 이해, 패턴 적용, 서술형 취약도, 반복 오답 문법이 표시된다.

### 영어 쓰기 기본 흐름

1. 수준, 주제, 쓰기 유형을 선택한다.
2. 사용할 핵심 단어와 문장 틀을 확인한다.
3. 문장 완성 또는 문장 배열로 기본 구조를 연습한다.
4. 짧은 문장이나 문단을 작성한다.
5. 문법, 어휘, 어순, 연결어를 점검한다.
6. 교사 첨삭을 받고 다시 쓴다.
7. 최종 글을 포트폴리오에 저장한다.
8. 교사용 화면에는 문장 정확성, 어휘 사용, 글 구조, 다시 쓰기 완료율이 표시된다.

### 내신 대비반 기본 흐름

1. 학년, 출판사, 교과서 판본, Lesson을 선택한다.
2. 본문을 문단별로 나누고 핵심 흐름과 주제를 확인한다.
3. 본문 단어와 숙어를 뜻, 품사, 문맥상 의미로 분석한다.
4. 본문 핵심 문법을 예문과 함께 정리한다.
5. 문장별로 주어, 동사, 목적어, 보어, 수식어, 접속 구조를 분석한다.
6. 직독직해, 자연스러운 해석, 시험 포인트를 함께 확인한다.
7. 학교 시험형 문제와 서술형 문제를 풀고 오답을 본문 위치와 연결한다.
8. 교사용 화면에는 출판사별 진도, Lesson별 취약 본문, 문법·단어·문장 분석 성취도가 표시된다.

### 리딩반 기본 흐름

1. 학년군, 난이도, 장르, 독해 스킬을 선택한다.
2. 지문 제목과 주제어를 먼저 보고 예측한다.
3. 핵심 어휘와 표현을 확인한다.
4. 문단별 중심 내용을 표시한다.
5. 세부 내용, 추론, 순서, 요약, 목적 파악 문제를 푼다.
6. 정답 근거 문장을 지문에서 확인한다.
7. 후속 요약 쓰기 또는 짧은 의견 쓰기를 진행한다.
8. 교사용 화면에는 독해 속도, 정답 근거 찾기, 중심 내용 파악, 추론 취약도가 표시된다.

### 영어 동화책 기본 흐름

1. 수준, 주제, 등장인물 수에 맞는 동화책을 고른다.
2. 표지와 장면 그림을 보고 이야기를 예측한다.
3. 장면별 읽어주기 음성을 듣고 따라 읽는다.
4. 등장인물 역할을 나누어 짧은 역할극을 한다.
5. 핵심 단어와 반복 문장 패턴을 확인한다.
6. 이야기 순서 맞추기와 장면 이해 질문을 푼다.
7. 다시 말하기, 그림 설명, 짧은 독후활동을 작성한다.
8. 교사용 화면에는 읽기 참여도, 역할극 참여도, 이야기 이해도, 다시 말하기 성취도가 표시된다.

### 영어 도서관 기본 흐름

1. 학생 수준, 관심 주제, 읽기 목표에 맞는 도서를 고른다.
2. 표지, 제목, 핵심 단어를 보고 내용을 예측한다.
3. 오디오가 있으면 듣기와 읽기를 함께 진행한다.
4. 읽은 날짜, 읽은 시간, 읽은 쪽수 또는 완료 여부를 기록한다.
5. 핵심 어휘, 이해 질문, 좋아한 장면을 정리한다.
6. 독서 기록장 또는 짧은 독후활동을 작성한다.
7. 교사는 학생별 다독량, 선호 장르, 이해도, 독서 지속성을 모니터링한다.

1. 원본 자료를 `content/*/source/`에 넣는다.

4. 유치원 활동 그림카드 누락, 영어 노래방 가사 타임라인 누락, 듣기 음원·대본 누락, 말하기 프롬프트 누락, 파닉스 음가 누락, 읽기 유창성 문장 누락, 초등 교과서 Lesson 메타데이터 누락, 오디오 키 누락, 중복 단어, 빈 뜻, 빈 예문, 쓰기 과제 누락, 품사 누락, 학년군 누락, 문법 정답 누락, 교과서 판본 누락, 본문 분석 누락, 리딩 정답 근거 누락, 동화책 장면 정보 누락, 도서관 메타데이터 누락을 검사한다.
5. 앱 DB 반영 후 원본 수와 반영 수를 대조한다.
6. 누락·중복·수정 항목을 `content/*/audits/`에 남긴다.

교육청 필수 영단어 자료는 지역과 연도에 따라 달라질 수 있으므로, 실제 입력 전에는 반드시 최신 출처와 사용 가능 범위를 확인한다.

## 구현 순서

0. 미국·영국 전문가 기준 매핑표를 먼저 만든다.
   Common Core, ACTFL, WIDA, National Curriculum, Reading Framework, CEFR형 성취문을 프로젝트의 `skillStrand`, `communicationMode`, `cefrCanDo`, `evidenceType` 필드와 연결한다.

1. TESOL 수업 설계 매핑표를 만든다.
   TESOL 6 Principles, TESOL teacher preparation standards, CELTA형 topic areas를 `learnerProfile`, `languageAnalysis`, `tesolStage`, `interactionPattern`, `assessmentEvidence`, `teacherReflection` 필드와 연결한다.

2. 영어 콘텐츠 DB 카탈로그를 만든다.
   유치원 영어반, 영어 노래방, 영어 듣기, 영어 말하기, 파닉스 교실, 영어 읽기, 초등 영어 교과서, 초등 영단어, 초등 영문법, 영어 쓰기, 중등 영단어, 중등 영문법, 내신 대비반, 리딩반, 영어 동화책, 영어 도서관 열여섯 개의 콘텐츠 DB 메타데이터를 먼저 등록한다.

3. TESOL 공통 Lesson 템플릿을 먼저 구현한다.
   diagnose, prepare, notice, practice, communicate, feedback, reflect 단계가 모든 트랙에서 재사용되도록 공통 화면과 데이터 구조를 만든다.

4. 유치원 영어반 샘플 10개 테마를 넣는다.
   인사, 색깔, 숫자, 동물, 가족, 음식, 감정, 교실 표현을 그림카드, 음성, 챈트, 몸동작 활동으로 먼저 완성한다.

5. 영어 노래방 샘플 10곡을 넣는다.
   자체 제작 노래 또는 라이선스 확인된 노래로 가사 하이라이트, 따라 부르기, 발음·리듬 포인트를 먼저 완성한다.

6. 영어 듣기 샘플 20개 음원을 넣는다.
   대화 듣기, 핵심 정보 찾기, 받아쓰기, 섀도잉, 정답 근거 확인 흐름을 먼저 완성한다.

7. 영어 말하기 샘플 20개 상황을 넣는다.
   따라 말하기, 역할극, 질문·응답, 짧은 발표, 녹음 피드백 흐름을 먼저 완성한다.

8. 파닉스 교실 샘플 10개 단원을 넣는다.
   알파벳 음가, 단모음, 기본 자음, CVC 블렌딩, 짧은 단어 읽기까지 먼저 완성한다.

9. 영어 읽기 샘플 20개 문장·문단을 넣는다.
   모범 읽기, 끊어 읽기, 시간 읽기, 녹음, 유창성 피드백 흐름을 먼저 완성한다.

10. 초등 영어 교과서 샘플 6개 Lesson을 넣는다.
   학년, 출판사, 단원 목표, 의사소통 표현, 듣기·말하기·읽기·쓰기 활동을 먼저 완성한다.

11. 초등 영단어 샘플 20개 단원을 넣는다.
   단어 카드, 뜻 맞히기, 철자 입력, 오답 노트, 누적 복습까지 먼저 완성한다.

12. 초등 영문법 샘플 10개 단원을 넣는다.
   개념 카드, 문장 배열, 빈칸, 오류 수정 문제를 만든다.

13. 영어 쓰기 샘플 20개 과제를 넣는다.
   단어 쓰기, 문장 완성, 문단 쓰기, 첨삭, 다시 쓰기 흐름을 먼저 완성한다.

14. 중등 영단어와 중등 영문법을 같은 스키마로 확장한다.
   중등은 내신형 문항과 예문 독해 비중을 높인다.

15. 영어 리딩반 샘플 10개 지문을 넣는다.
   중심 내용, 세부 내용, 추론, 요약, 정답 근거 찾기 문제를 먼저 완성한다.

16. 영어 내신 대비반 샘플을 넣는다.
   8종 교과서별 출판사·판본 메타데이터를 먼저 만들고, 본문 분석, 문법 분석, 단어 분석, 문장 분석 레이어를 연결한다.

17. 영어 동화책 샘플 10권을 넣는다.
   자체 제작 또는 라이선스 확인된 동화책으로 장면 읽기, 역할극, 이야기 순서, 독후활동을 먼저 완성한다.

18. 영어 도서관 샘플 30권 메타데이터를 넣는다.
   유치원 그림책, 초급 리더스, 초등 리더스, 중등 리딩 도서로 나누고 읽기 기록, 독후활동, 교사 추천 흐름을 연결한다.

19. 교사용 모니터링과 리포트를 영어 지표에 맞춘다.
   음가 인식률, 블렌딩 정확도, 단어 암기율, 철자 정확도, 문법 개념 이해도, 문장 작성 안정성을 별도 지표로 둔다.

20. 관리자 승인과 권한 관리를 기존 수학 전자북 루틴과 동일하게 연결한다.
   교사 승인, 학생 승인, 접근 제한, 비밀번호 초기화, 데이터 백업을 유지한다.

21. 단일 HTML, 모바일, Windows 설치 산출물을 갱신한다.
   산출물 해시와 검증 결과를 문서에 기록한다.

앱 코드를 수정한 경우 기본 검증은 아래 순서로 진행한다.

```powershell
$env:Path='C:\Program Files\nodejs;' + $env:Path
npm run lint
npm run verify
npm run onefile
```

서버 DB 또는 학습 기록을 수정한 경우 추가로 확인한다.

```powershell
npm run pages:deploy
```

```powershell
npm run installer:win
```

콘텐츠만 추가한 경우에는 다음 항목을 별도 확인한다.

- 원본 항목 수와 앱 반영 항목 수가 일치하는지 확인한다.
- 콘텐츠별 `learningObjective`, `skillStrand`, `cefrCanDo`, `englishVariant`, `sourceLicenseStatus`, `evidenceType`이 비어 있지 않은지 확인한다.
- TESOL 필드인 `tesolStage`, `languageAnalysis`, `interactionPattern`, `scaffolding`, `ccqIcq`, `reflectionPrompt`가 필요한 활동에 연결되어 있는지 확인한다.
- 각 Lesson이 diagnose, prepare, notice, practice, communicate, feedback, reflect 단계 중 최소 5단계 이상을 갖는지 확인한다.
- 핵심 언어 항목에 form, meaning, use, pronunciation, common errors가 저장되어 있는지 확인한다.
- 미국식·영국식 철자, 발음, 표현 차이가 필요한 항목에 `englishVariant`와 `accentAudioKey`가 연결되어 있는지 확인한다.

## UI 설계 체크포인트

- 학생 첫 화면은 학습 시작 버튼과 현재 단원 선택이 먼저 보여야 한다.
- 관리 기능은 상단 메뉴바나 모달 안으로 숨기고 학습 콘텐츠를 가리지 않는다.
- 유치원 영어반은 큰 그림카드, 큰 듣기 버튼, 짧은 챈트, 몸동작 안내가 중심이어야 한다.
- 영어 노래방은 가사 하이라이트, 큰 재생 버튼, 느린 속도, 원속도, 반주 모드, 녹음 체크가 한 화면에 있어야 한다.
- 영어 듣기는 전체 듣기, 다시 듣기, 받아쓰기, 섀도잉, 정답 근거 확인이 한 흐름 안에 있어야 한다.
- 영어 말하기는 예시 답변, 문장 틀, 역할극, 녹음, 교사 피드백이 한 화면 흐름 안에 있어야 한다.
- 파닉스 학습은 소리 듣기, 글자 고르기, 블렌딩, 따라 말하기가 한 화면 흐름 안에서 이어져야 한다.
- 영어 읽기는 끊어 읽기 표시, 모범 읽기, 시간 읽기, 녹음, 유창성 피드백이 한 화면 흐름 안에 있어야 한다.
- 초등 영어 교과서는 Lesson 선택, 핵심 표현, 대화 듣기, 말하기 활동, 읽기·쓰기 활동이 한 단원 안에서 이어져야 한다.
- 단어 학습은 카드 넘김, 듣기, 뜻 확인, 철자 입력이 한 손으로도 가능해야 한다.
- 영어 쓰기는 작성 영역, 문장 틀, 체크리스트, 첨삭, 다시 쓰기 버튼이 같은 과제 안에서 이어져야 한다.
- 문법 학습은 개념 설명과 문제 풀이가 같은 화면에서 끊기지 않아야 한다.
- 내신 대비반은 본문, 문법, 단어, 문장 분석 탭을 같은 Lesson 안에서 빠르게 오갈 수 있어야 한다.
- 리딩반은 지문, 문제, 정답 근거, 요약 입력이 같은 흐름 안에서 이어져야 한다.
- 영어 동화책은 장면 넘김, 읽어주기, 역할극, 이야기 순서, 독후활동이 같은 책 안에서 이어져야 한다.
- 영어 도서관은 레벨별 서가, 주제 필터, 읽기 기록장, 독후활동, 교사 추천 도서가 분리되어야 한다.
- 교사용 화면은 학생별 비교와 반복 오답 확인이 빠르게 가능해야 한다.
- 모바일에서는 왼쪽 패널을 하단 탭이나 접이식 패널로 바꾼다.
- 버튼은 기능이 명확한 아이콘과 텍스트를 함께 쓰고, 같은 동작은 같은 위치에 둔다.
- 텍스트 크기는 viewport 연속 스케일보다 CSS 변수와 브레이크포인트로 조정한다.

## 교사용 운영 대시보드 설계

교사용 화면은 단순 점수판이 아니라 수업 결정을 빠르게 돕는 운영 화면이어야 한다. 첫 화면에는 오늘 수업에 필요한 학생, 과제, 취약도, 추천 루틴이 바로 보이게 한다.

| 영역 | 표시 정보 | 교사 액션 |
| --- | --- | --- |
| 오늘 수업 큐 | 학생별 오늘 추천 트랙, 미완료 과제, 내신 일정, 재도전 항목 | 바로 배정, 오늘 제외, 난이도 조정 |
| 기능별 프로필 | 듣기, 말하기, 읽기, 쓰기, 파닉스, 어휘, 문법, 독서 습관 점수 | 취약 기능 보충반 배정 |
| 파닉스·읽기 결손 | 혼동 음가, decoding 실패 단어, read-aloud 정확도, 재읽기 필요 문장 | 파닉스 보충 단원 배정 |
| 교과·내신 관리 | 학년, 출판사, Lesson, 본문 분석률, 서술형 취약 포인트 | Lesson 복습, 변형 문제 배정 |
| 산출물 포트폴리오 | 말하기 녹음, 받아쓰기, 쓰기 초안, 첨삭본, 독후활동 | 학부모 상담 자료 생성 |
| 미국·영국 변이 검토 | variant가 필요한 단어, 발음 비교 항목, 교사용 주석 | 학생 보기 공개 여부 선택 |
| 리포트 발행 | 일간, 주간, 월간, 학부모 상담서, 내신 대비 리포트 | PDF 저장, 출력, 공유 |

학생 상세 화면에는 `다음 15분 수업 추천`, `이번 주 보충 루틴`, `집에서 할 5분 과제`, `교사 코멘트`를 둔다. 이렇게 하면 전자북이 단순 문제풀이 도구가 아니라 학원형 자기주도 수업 운영 시스템으로 동작한다.

## 보고서 설계

영어 전자북 리포트는 수학 점수 중심 보고서와 다르게 파닉스, 어휘, 문법 능력 지표를 분리한다.

| 보고서 항목 | 파닉스 지표 | 영단어 지표 | 영문법 지표 |
| --- | --- | --- | --- |
| 오늘 학습 | 학습 음가 수, 블렌딩 성공률 | 학습 단어 수, 암기 성공률 | 풀이 문항 수, 개념 정답률 |
| 오답 분석 | 혼동 음가, 읽기 오류 패턴 | 뜻 혼동, 철자 오류, 예문 이해 | 시제, 품사, 어순, 문장 구조 |
| 누적 복습 | 반복 취약 음가, 장기 미복습 패턴 | 반복 오답 단어, 장기 미복습 단어 | 반복 취약 문법, 재학습 필요 단원 |
| 교사 코칭 | 소리 따라 하기, 블렌딩 과제 | 단어장 추천, 받아쓰기 과제 | 문장 쓰기 과제, 오류 수정 과제 |
| 학부모 상담 | 발음 자신감, 문자·소리 연결 습관 | 어휘 습득 습관, 복습 주기 | 문장 이해와 표현 안정성 |

유치원 영어반과 영어 도서관은 별도 지표를 추가한다.

| 보고서 항목 | 유치원 영어반 지표 | 영어 도서관 지표 |
| --- | --- | --- |
| 오늘 학습 | 참여 활동 수, 따라 말하기 반응, 그림카드 인지 | 읽은 도서 수, 읽은 시간, 완료 도서 |
| 오답 분석 | 헷갈린 그림카드, 반응이 약한 표현 | 이해 질문 오답, 어려운 어휘, 중도 중단 도서 |
| 누적 복습 | 자주 반복할 생활 표현, 가정 미션 수행 | 누적 독서량, 선호 장르, 미복습 도서 |
| 교사 코칭 | 챈트 반복, 몸동작 활동, 가정 연계 표현 | 레벨 조정, 추천 도서, 독후활동 피드백 |
| 학부모 상담 | 영어 노출 태도, 말하기 자신감, 가정 복습 습관 | 독서 습관, 흥미 주제, 원서 읽기 지속성 |

영어 노래방과 영어 동화책은 별도 지표를 추가한다.

| 보고서 항목 | 영어 노래방 지표 | 영어 동화책 지표 |
| --- | --- | --- |
| 오늘 학습 | 부른 노래 수, 반복 횟수, 녹음 완료 | 읽은 장면 수, 듣기 완료, 역할극 참여 |
| 오답 분석 | 약한 발음, 놓친 가사, 리듬 취약 구간 | 어려운 장면, 헷갈린 등장인물, 이해 질문 오답 |
| 누적 복습 | 반복 추천곡, 미완료 녹음, 표현 복습 | 다시 읽을 장면, 미완료 독후활동, 반복 어휘 |
| 교사 코칭 | 발음·리듬 코칭, 자신감 회복 노래, 율동 과제 | 역할극 과제, 다시 말하기, 장면 설명 과제 |
| 학부모 상담 | 말하기 자신감, 영어 노출 태도, 가정 노래 복습 | 이야기 이해력, 표현력, 독후활동 습관 |

| --- | --- | --- | --- | --- |
| 오늘 학습 | 들은 음원 수, 듣기 정답률, 받아쓰기 완료 | 녹음 수, 역할극 완료, 발표 완료 | 읽은 문장 수, 시간 읽기 완료, 녹음 완료 | 작성 문장 수, 문단 수, 다시 쓰기 완료 |
| 오답 분석 | 놓친 핵심 정보, 받아쓰기 오류 | 발음, 문장 정확성, 응답 지연 | 발음 오류, 끊어 읽기 오류, 속도 저하 | 어순, 시제, 철자, 연결어 오류 |
| 누적 복습 | 반복 취약 음원, 자주 놓친 표현 | 반복 말하기 프롬프트, 자신감 낮은 주제 | 반복 읽기 문장, 유창성 취약 구간 | 반복 오류 패턴, 미완료 다시 쓰기 |

| 학부모 상담 | 듣기 집중도, 소리 인식 습관 | 말하기 자신감, 표현 사용도 | 읽기 정확도, 읽기 속도, 자신감 | 글쓰기 구조, 문장 정확도, 표현 성장 |

초등 영어 교과서는 별도 지표를 추가한다.

| 보고서 항목 | 초등 영어 교과서 지표 |

| 보고서 항목 | 내신 대비반 지표 | 리딩반 지표 |
| --- | --- | --- |
| 오늘 학습 | 교과서, Lesson, 본문 분석 완료율 | 지문 수, 독해 문제 수, 정답률 |
| 오답 분석 | 본문 위치, 문법 포인트, 단어·숙어 취약도 | 중심 내용, 세부 내용, 추론, 요약 취약도 |
| 누적 복습 | 출판사별 취약 Lesson, 반복 오답 문장 | 반복 취약 독해 스킬, 장기 미복습 지문 |
| 교사 코칭 | 본문 암기 과제, 문장 분석 과제, 서술형 대비 | 독해 속도, 근거 찾기, 요약 쓰기 과제 |
| 학부모 상담 | 학교 시험 대비 현황, 교과서별 보완 계획 | 읽기 습관, 지문 이해력, 추론력 성장 기록 |

전문가 기준 리포트는 모든 트랙에 공통으로 붙인다.

| 공통 리포트 항목 | 표시 방식 |
| --- | --- |
| `can-do` 성취문 | 학생이 이번 주 실제로 할 수 있게 된 영어 행동을 한 문장으로 표시한다. |
| 기능별 균형 | 듣기, 말하기, 읽기, 쓰기, 언어지식, 파닉스, 독서 활동 시간을 시각화한다. |
| 학습 증거 | 퀴즈 점수, 녹음, 받아쓰기, 쓰기 초안, 첨삭본, 독후활동을 날짜별로 묶는다. |
| 재학습 원인 | 개념 미이해, 어휘 부족, 음가 혼동, 속도 문제, 집중도 문제, 시험유형 미숙을 구분한다. |
| 미국·영국 변이 노출 | 학습자가 접한 발음·철자 변이를 기록하되 평가 점수로 벌점화하지 않는다. |
| 교사 다음 조치 | 보충 단원, 재녹음, 다시 읽기, 다시 쓰기, 내신 변형 문제를 추천한다. |

리포트는 일간, 주간, 월간으로 생성하고 교사·학생별로 DB에 기록한다. PDF 저장, PC 출력, 카카오톡 공유, 이메일 전송은 교사용 또는 관리자 권한에서만 제공한다.

## 완료 기준

- 열여섯 개 영어 콘텐츠 DB가 콘텐츠 카탈로그에서 분리되어 보인다.
- 미국식 표준 기반 문해력, 영국식 파닉스·읽기 유창성, CEFR형 `can-do` 성취문이 콘텐츠 메타데이터와 리포트에 연결되어 있다.
- TESOL 6 Principles와 CELTA형 교사교육 주제가 학습자 분석, 언어 분석, 수업 단계, 피드백, 교사 성찰 필드로 연결되어 있다.
- 모든 Lesson이 diagnose, prepare, notice, practice, communicate, feedback, reflect 루틴으로 설계되어 있다.
- 교사용 수업안에서 main aim, subsidiary aim, anticipated problems, solutions, CCQ, ICQ, interaction pattern, feedback plan을 확인할 수 있다.
- 모든 콘텐츠 item에 학습 목표, 기능 축, 수준, 미국·영국 변이, 출처·라이선스 상태, 평가 증거가 저장되어 있다.
- 교사용 대시보드에서 학생별 기능 프로필, 파닉스·읽기 결손, 교과·내신 진도, 포트폴리오가 한 화면에 보인다.
- 유치원 영어반에서 그림카드, 챈트, 몸동작, 생활영어, 가정 미션이 분리되어 동작한다.
- 영어 노래방에서 가사 하이라이트, 따라 부르기, 발음·리듬 연습, 녹음 기록이 분리되어 동작한다.
- 영어 듣기에서 대화 듣기, 받아쓰기, 섀도잉, 정답 근거 확인이 분리되어 동작한다.
- 영어 말하기에서 따라 말하기, 역할극, 질문·응답, 발표, 녹음 피드백이 분리되어 동작한다.
- 파닉스 교실에서 알파벳, 음가, 블렌딩, 디코딩 활동이 분리되어 동작한다.
- 영어 읽기에서 소리내어 읽기, 끊어 읽기, 시간 읽기, 유창성 피드백이 분리되어 동작한다.
- 초등 영어 교과서에서 Lesson, 핵심 표현, 듣기·말하기·읽기·쓰기 활동이 분리되어 동작한다.
- 내신 대비반에서 8종 영어 교과서별 본문 분석, 문법 분석, 단어 분석, 문장 분석이 분리되어 동작한다.
- 리딩반에서 지문 읽기, 독해 문제, 정답 근거, 요약 활동이 분리되어 동작한다.
- 영어 동화책에서 장면별 듣기, 따라 읽기, 역할극, 이야기 순서, 독후활동이 분리되어 동작한다.
- 영어 도서관에서 레벨별 서가, 도서 메타데이터, 읽기 기록장, 독후활동이 분리되어 동작한다.
- 학생 학습 기록이 콘텐츠 업데이트 후에도 유지된다.
- 관리자, 교사, 학생 로그인 흐름이 수학 전자북과 같은 권한 기준으로 동작한다.
- 영단어와 영문법 학습 엔진이 서로 다른 활동 유형을 지원한다.
- 영어 쓰기에서 문장 만들기, 문단 쓰기, 첨삭, 다시 쓰기가 분리되어 동작한다.
- 교사용 모니터링에서 영어 전용 지표가 보인다.
- 원본 콘텐츠 수와 앱 반영 수를 대조한 검증 보고서가 있다.
- `npm run lint`, `npm run verify`, `npm run onefile`이 통과한다.
- 단일 HTML, 모바일 `study.html`, 서비스워커 캐시, Cloudflare Pages, Windows 설치 파일 갱신 여부가 기록되어 있다.
