# CODEX 수학 익힘북 전자북 · 5학년 3월~다음 해 2월

업로드한 5학년 3월~다음 해 2월 기초력 문제 플러스북 PDF 흐름을 바탕으로 만든 자동 채점 수학 연습 웹앱입니다.
Vite + React + TypeScript로 구성했습니다.

## 담은 내용

- **1단원 자연수의 혼합 계산** — 덧셈·뺄셈 / 괄호 / 곱셈·나눗셈 /
  사칙연산 혼합 / 연산 기호 찾기 / 거꾸로 계산
- **2단원 약수와 배수** — 약수 / 배수 / 공약수 / 공배수 /
  최대공약수 / 최소공배수 / 활용 문제
- **3단원 대응 관계** — 배 관계 / 차가 일정한 관계 / 표 완성 /
  대응 관계를 식으로 나타내기 / 규칙 배열 적용
- **4단원 약분과 통분** — 크기가 같은 분수 / 기약분수 / 통분 /
  두 분수·세 분수 비교 / 분수와 소수 비교
- **5단원 분수의 덧셈과 뺄셈** — 진분수 덧셈 / 대분수 덧셈 /
  진분수 뺄셈 / 받아내림 대분수 뺄셈 / 빈칸 분수 / 수 카드 문제

  평행사변형 / 삼각형 / 사다리꼴 / 복합 도형 넓이
- **7단원 수의 범위와 어림하기** — 이상·이하 / 초과·미만 /
  십의 자리·백의 자리 반올림 / 어림셈
- **8단원 분수의 곱셈** — 분수 × 자연수 / 자연수 × 분수 /
  분수 × 분수 / 대분수 곱셈 / 전체의 분수만큼
- **9단원 합동과 대칭** — 대응변 / 대응각 / 대칭축 /
  점대칭도형 / 좌표 대칭
- **10단원 소수의 곱셈** — 소수 × 자연수 / 자연수 × 소수 /
  소수 × 소수 / 소수 길이 넓이 / 빈칸 곱셈
- **11단원 직육면체** — 모서리 길이의 합 / 면·모서리·꼭짓점 /
  한 면의 넓이 / 쌓기나무 / 평행한 면
- **12단원 평균과 가능성** — 평균 / 빠진 값 / 전체 합 /
  가능성 표현 / 가능성 비교
- **13단원 5학년 총정리** — 혼합 계산 / 약수와 배수 / 분수 /
  넓이 / 분수·소수 누적 복습
- **14단원 6학년 준비 학습** — 분수 나눗셈 / 소수 나눗셈 /
  비 / 그래프 읽기 / 부피 맛보기

문항은 PDF의 고정 문제를 그대로 옮기지 않고 **같은 유형을 난수로 자동 생성**해,
같은 내용을 무한 반복 연습할 수 있습니다. 나눗셈은 항상 나누어떨어지고
분수 문항은 입력 형식과 공통분모 조건까지 채점합니다.

## 주요 기능

- 정답 입력 시 **즉시 자동 채점** (O/X)
- 오답이면 **정답과 한 줄 풀이** 표시
- 단원별 **스토리 지도**, 캐릭터 대화, 미션 카드, 모험 힌트 제공
- 전자북 리더형 개인화: **야간 모드 / 집중 모드 / 큰 글자 / TTS 낭독 / 복습 저장**
- 누적 학습량, 정답률, 오답률, 단원별 진행률, 복습 저장 수를 로컬에 보존
- 이번 세션 **정답 수·정답률** 표시
- 단원별 유형 선택 또는 "3월~다음 해 2월 전체 학습"
- 선택형 대응식, 분수 입력, 통분 쌍 입력, 비교 기호 입력
- Enter 키로 채점 → 다음 문제 이동
- 세부 유형마다 10문제씩 구성되는 세션 진행률
- 모바일/PC 반응형 전자북 대시보드

## 실행 방법

```bash
cd app
npm install      # 최초 1회
npm run dev      # 개발 서버 (http://localhost:5173)
```

배포용 정적 파일을 만들려면:

```bash
npm run build    # 타입체크 + 빌드 → dist/
npm run preview  # 빌드 결과 미리보기
```

단일 HTML 파일로 묶으려면:

```bash
npm run onefile
```

결과물은 `../CODEX 수학 익힘북 전자북(26.05.17)/CODEX-수학-익힘북-전자북.html`에 생성됩니다.

## Cloudflare Pages 배포

이 프로젝트는 별도 Node 서버 없이 Cloudflare Pages 정적 호스팅으로 운영합니다.

Cloudflare Pages에서 프로젝트 루트를 현재 폴더로 잡는 경우:

```bash
cd app && npm ci && npm run pages:build
```

빌드 출력 디렉터리는 `app/dist`입니다.

Cloudflare Pages에서 프로젝트 루트를 `app` 폴더로 잡는 경우:

```bash
npm ci && npm run pages:build
```

빌드 출력 디렉터리는 `dist`입니다.

루트 `wrangler.toml`은 직접 배포나 CI 배포에서 `app/dist`를 Pages 산출물로 사용하도록 고정합니다. `app/public/_headers`는 빌드 때 `dist/_headers`로 복사되어 Cloudflare Pages에서 다음 운영 정책을 적용합니다.

- 해시가 붙은 `/assets/*` 번들은 1년 immutable 캐시로 재방문 로딩을 줄입니다.
- `/`와 `/*.html`은 즉시 재검증하도록 두어 새 배포가 빠르게 반영됩니다.
- 기본 보안 헤더를 정적 응답 전체에 적용합니다.
- Node 런타임은 Cloudflare Pages v3 기본값인 `22.16.0`으로 고정합니다.

로컬에서 Cloudflare Pages와 비슷하게 확인하려면:

```bash
npm run pages:preview
```

로그인한 Wrangler 환경에서 직접 배포하려면:

```bash
npm run pages:deploy
```

이 명령은 Cloudflare Pages production 브랜치인 `main`으로 배포합니다.
