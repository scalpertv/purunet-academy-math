# AGENTS.md

이 프로젝트의 기본 작업 기준은 [CLAUDE.md](CLAUDE.md)입니다.

## 우선순위

1. 시스템, 개발자, 사용자 지시를 먼저 따른다.
2. 그다음 이 `AGENTS.md`를 따른다.
3. 세부 작업 방식은 `CLAUDE.md`를 기준으로 병합한다.

## 작업 전 확인

- 비자명한 작업은 바로 코딩하지 않고, 짧은 계획과 검증 기준을 먼저 세운다.
- 기존 `checklist.md`와 `context-notes.md`가 있으면 이어서 갱신한다.
- 없으면 작업 시작 전에 새로 만든다.
- 불명확한 요구는 임의로 확장하지 말고, 가정과 선택지를 드러낸다.

## 코드 수정 원칙

- 요청과 직접 연결되는 파일만 수정한다.
- 기존 스타일과 구조를 우선한다.
- 단일 사용 목적의 과한 추상화는 만들지 않는다.
- 새 소스 파일을 만들 때는 첫 줄에 파일 역할을 설명하는 한국어 한 줄 주석을 둔다.
- 한국어 답변은 문장 끝을 콜론으로 끝내지 않는다.

## 검증 원칙

- 코드를 수정했다면 완료 전에 가능한 검증을 실행한다.
- 테스트가 없으면 최소한 빌드, 타입 검사, 문법 검사 중 가능한 것을 실행한다.
- 검증 실패는 실제 오류 문구를 읽고 원인을 확인한 뒤 수정한다.

## 현재 전자북 프로젝트 검증

이 프로젝트에서 앱 코드를 수정하면 기본 검증은 아래 순서로 실행한다.

```powershell
$env:Path='C:\Program Files\nodejs;' + $env:Path
npm run lint
npm run verify
npm run onefile
```

`npm`이 인식되지 않으면 먼저 `C:\Program Files\nodejs`를 `PATH` 앞에 붙인다.

단일 HTML을 갱신해야 하는 변경이면 생성된 파일을 아래 위치에 반영한다.

- `푸르넷수학-연습.html`
- `모바일 홈페이지형 전자북(26.05.17)/study.html`

모바일 파일을 갱신했다면 `모바일 홈페이지형 전자북(26.05.17)/sw.js`의 캐시 버전도 올린다.

## Git 처리

- Git 저장소라면 의미 단위로 커밋한다.
- Git 저장소가 아니면 커밋 대신 변경 파일, 검증 명령, 검증 결과를 최종 보고에 남긴다.
