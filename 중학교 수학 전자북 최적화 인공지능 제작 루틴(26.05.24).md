수학 PDF를 단순 변환이 아니라 **인터랙티브 전자북 + 학습 플랫폼형 콘텐츠**로 제작하려면, 핵심은 아래 5가지를 동시에 해결하는 것입니다.

1. 수식 깨짐 방지
2. 필기·그래프·도형 유지
3. 모바일 최적화
4. 문제 풀이 인터랙션
5. AI 자동화 제작 파이프라인

수학은 일반 텍스트 전자책과 완전히 다르게 접근해야 합니다.

---

# 수학 PDF 전자북 제작의 최적 구조

## 추천 최종 구조

```text
PDF 원본
↓
OCR + 수식 추출
↓
구조화 XML/HTML5 변환
↓
MathJax/KaTeX 수식 렌더링
↓
인터랙티브 문제 추가
↓
PWA/앱/웹 전자북 배포
```

---

# 1단계 — PDF를 직접 EPUB 변환하지 말 것

가장 많이 실패하는 방식:

```text
PDF → EPUB 자동변환
```

문제점:

* 수식 깨짐
* 도형 위치 틀어짐
* 표 레이아웃 붕괴
* 분수/루트 깨짐
* 이미지화되어 검색 불가
* 모바일에서 확대 축소 지옥

수학은 PDF 기반 좌표 문서라서 일반 EPUB 변환이 거의 망가집니다.

---

# 가장 좋은 방식

## PDF → HTML5 기반 전자북

이 방식이 현재 가장 안정적입니다.

### 이유

| 항목     | HTML5 |
| ------ | ----- |
| 수식 표현  | 매우 강력 |
| 모바일 대응 | 최고    |
| 인터랙션   | 가능    |
| AI 연동  | 쉬움    |
| 웹앱화    | 가능    |
| 음성/TTS | 가능    |
| 문제풀이   | 가능    |
| 애니메이션  | 가능    |

---

# 2단계 — OCR + 수식 추출

수학은 일반 OCR만 쓰면 안 됩니다.

## 추천 엔진

### 1순위

| 도구        | 특징           |
| --------- | ------------ |
| Mathpix   | 수식 OCR 최강    |
| Nougat    | 무료 AI 논문 OCR |
| PaddleOCR | 한글 강력        |
| Tesseract | 무료 기본 OCR    |

---

# 가장 추천하는 실전 조합

## 수학 전용

```text
PDF
↓
Mathpix
↓
LaTeX + Markdown 추출
↓
HTML5 변환
```

이 방식이 수식 보존율이 가장 높습니다.

---

# 3단계 — 수식은 반드시 LaTeX 기반으로 관리

수학 전자북 핵심입니다.

## 잘못된 방식

```text
수식을 이미지로 저장
```

문제:

* 확대 시 깨짐
* 검색 안됨
* AI 분석 불가
* 모바일 느림

---

## 올바른 방식

```text
LaTeX 수식
+
MathJax/KaTeX 렌더링
```

---

# 추천 렌더링 엔진

## 1순위

| 엔진      | 특징     |

| KaTeX   | 매우 빠름  |

# EPUB3가 좋은 이유

수학 전자북은 EPUB2가 아니라 EPUB3 사용.

이유:

* MathML 지원
* 오디오 가능
* JS 가능
* 인터랙션 가능
